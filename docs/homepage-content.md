## AI가 코드를 작성하는 시대, 사람은 무엇을 해야 할까요?

우리는 "사주풀이"라는 전통 명리학 분석 웹앱을 **AI 에이전트가 직접 코드를 작성하는 방식**으로 개발했습니다. 기획부터 설계, 구현, 테스트, 배포까지 — 전 과정에서 AI와 사람이 어떻게 협업하는지를 실증한 프로젝트입니다.

**라이브 서비스**: [saju-app-iota.vercel.app](https://saju-app-iota.vercel.app)
**소스 코드**: [github.com/ORN-Consulting/SaJu](https://github.com/ORN-Consulting/SaJu)

## 이 프로젝트가 보여주는 것

### 1. AI 에이전트 협업 개발 (Harness Engineering)

이 프로젝트에서 **사람이 한 일**과 **AI가 한 일**은 명확히 구분됩니다.

| 사람이 한 일 | AI가 한 일 |
|---|---|
| 프로젝트 철학과 원칙 수립 | TypeScript 타입 시스템 설계 및 구현 |
| 비즈니스 요구사항 정의 | 13개 매핑 테이블 + 계산 함수 작성 |
| 명리학 도메인 규칙 결정 | 5개 계산 엔진 모듈 구현 |
| 설계 문서 작성 및 리뷰 | 13개 UI 컴포넌트 개발 |
| AI 행동 규칙(하네스) 설계 | 26개 테스트 케이스 작성 및 통과 |
| 품질 검증 및 방향 조정 | Git 커밋, 빌드, 배포 수행 |

핵심은 **"사람이 설계하고, AI가 구현한다"**는 것입니다. 이를 가능하게 한 것이 **하네스 엔지니어링(Harness Engineering)** — AI 에이전트가 올바르게 동작하도록 규칙, 구조, 컨텍스트를 설계하는 기술입니다.

### 2. 하네스 엔지니어링이란?

AI에게 "사주 앱을 만들어라"라고 말하면 제대로 된 결과가 나오지 않습니다. 하지만 아래와 같은 **구조화된 지시 체계**를 갖추면, AI는 시니어 개발자 수준의 코드를 작성합니다.

```
하네스 구조:

CLAUDE.md          → AI가 세션 시작 시 자동으로 읽는 진입점
TASKS.md           → 18개 작업으로 분해된 실행 계획 + 완료 조건
AGENTS.md          → AI의 자율 범위, 코딩 규칙, 금지 사항
ARCHITECTURE.md    → 시스템 구조, 모듈 설계, 타입, 데이터 흐름
docs/design-docs/  → "왜 이렇게 만드는가"에 대한 결정 근거
docs/product-specs/→ "무엇을 만드는가"에 대한 상세 명세
```

이 하네스가 있으면 AI 에이전트는:
- 세션이 끊겨도 **이전 작업을 이어서** 진행합니다
- 명세에 없는 기능을 **임의로 추가하지 않습니다**
- 도메인 규칙(명리학)을 **정확히 준수**합니다
- 코드 스타일, 커밋 메시지, 테스트 규칙을 **일관되게 따릅니다**

## 개발 워크플로우

### 전체 흐름: 아이디어에서 배포까지

```
[1단계] 기획 · 설계
  │  프로젝트 철학 수립 (core-beliefs.md)
  │  기술 스택 선택 근거 문서화 (tech-stack-decisions.md)
  │  UI/UX 설계 결정 (ui-design-decisions.md)
  │  도메인 규칙 정의 (saju-engine-design.md)
  │  화면별 기능 명세 작성 (4개 문서)
  │
  ▼
[2단계] 하네스 구축
  │  CLAUDE.md — AI 진입점
  │  TASKS.md — 작업 분해 (4 Phase, 18 Task)
  │  AGENTS.md — AI 행동 규칙
  │  검증 데이터셋 7건 준비
  │  해석 텍스트 콘텐츠 작성
  │
  ▼
[3단계] AI 에이전트 실행
  │  Phase 0: 인프라 (git, Next.js, 디렉토리 구조)
  │  Phase 1: 데이터 레이어 (타입, 상수, 에러, 해석 텍스트)
  │  Phase 2: 계산 엔진 (만세력, 오행, 십성, 용신) + 26개 테스트
  │  Phase 3: UI (13개 컴포넌트, 반응형, 애니메이션)
  │  Phase 4: 통합 (페이지 조립, SEO, 빌드)
  │
  ▼
[4단계] 배포
  │  GitHub 저장소 생성 + push
  │  Vercel 연동 → 자동 배포
  │  라이브 서비스 가동
  │
  ▼
[운영] 지속적 개선
     새 세션 시작 → TASKS.md 확인 → 미완료 작업 이어서 진행
```

### 실제 소요 시간

| 단계 | 산출물 | 방식 |
|------|--------|------|
| 기획 · 설계 | 설계 문서 5개 + 기능 명세 5개 | 사람 (컨설턴트) |
| 하네스 구축 | CLAUDE.md, TASKS.md, AGENTS.md, 검증 데이터, 해석 텍스트 | 사람 + AI 협업 |
| 코드 구현 | 70+ 파일, 4,000+ 줄 코드, 26개 테스트 | AI 에이전트 (Claude Code) |
| 배포 | GitHub + Vercel | AI 에이전트 |

## 사용 기술 스택

### 프론트엔드 · 서비스

| 기술 | 역할 | 선택 근거 |
|------|------|----------|
| **Next.js 15** | React 기반 프레임워크 | SSG(정적 사이트 생성) 지원, Vercel 최적화 |
| **TypeScript** | 타입 안전 언어 | 10천간 × 12지지 × 5오행의 매핑 오류를 컴파일 타임에 방지 |
| **Tailwind CSS** | 유틸리티 CSS | 반응형 디자인을 클래스만으로 처리 |
| **lunar-javascript** | 만세력 라이브러리 | 음양력 변환, 절기 계산, 천간지지 지원 |

### 인프라 · 배포

| 기술 | 역할 | 비용 |
|------|------|------|
| **Vercel** | 호스팅 + CDN + 자동 HTTPS | 무료 티어 |
| **GitHub** | 소스 코드 관리 + CI/CD 트리거 | 무료 |

### AI · 개발 도구

| 기술 | 역할 |
|------|------|
| **Claude Code** (Anthropic) | AI 코딩 에이전트 — 코드 작성, 테스트, 커밋, 배포 |
| **하네스 엔지니어링** | AI 에이전트의 행동을 제어하는 문서 체계 설계 |
| **Vitest** | 테스트 프레임워크 — 26개 자동화 테스트 |

### 아키텍처 특징

```
┌─────────────────────────────────────────┐
│            Vercel (글로벌 CDN)            │
│                                         │
│   ┌───────────────────────────────┐     │
│   │     Next.js (정적 사이트)       │     │
│   │                               │     │
│   │   브라우저에서 모든 계산 수행     │     │
│   │   서버 통신 없음                │     │
│   │   DB 없음                      │     │
│   │   사용자 데이터 저장 없음        │     │
│   └───────────────────────────────┘     │
│                                         │
│   운영 비용: ₩0 (무료 티어)              │
└─────────────────────────────────────────┘
```

- **서버리스**: 모든 계산이 사용자 브라우저에서 완결
- **제로 코스트**: DB, 서버, API 키 없이 운영
- **개인정보 보호**: 생년월일시를 서버에 전송하지 않음

## 서비스 기능

### 사주팔자 분석 파이프라인

사용자가 생년월일시를 입력하면, 4단계 계산 파이프라인이 브라우저에서 즉시 실행됩니다.

```
입력 (생년월일시 + 성별)
  → 만세력 계산 (절기 기반 사주팔자 산출)
  → 오행 분석 (목·화·토·금·수 분포)
  → 십성 산출 (일간 기준 관계 분석)
  → 용신 판단 (억부법 기반 균형 분석)
  → 종합 해석 (규칙 기반 텍스트 조합)
```

### 주요 화면

| 화면 | 설명 |
|------|------|
| **랜딩** | 서비스 소개, 특징 카드 3개, CTA |
| **입력** | 양력/음력 선택, 연·월·일 드롭다운, 시진/정확시간 입력, 성별 |
| **사주 카드** | 4주(시·일·월·년) 카드 — 천간·지지·오행 색상·십성 표시 |
| **오행 차트** | 5가지 오행의 강약을 바 차트로 시각화 |
| **십성 분석** | 등장 십성의 의미를 친근한 문체로 해석 |
| **용신 결과** | 신강/신약 판정 + 용신·희신·기신 + 종합 해석 |

### AI 시대의 개발 방법론

이 프로젝트는 **"AI가 코드를 대체한다"**가 아니라, **"사람의 설계 능력이 AI의 생산성을 결정한다"**는 것을 보여줍니다.

- 잘 설계된 하네스 → AI가 시니어 개발자 수준의 코드를 생산
- 모호한 지시 → AI가 임의 판단, 품질 저하
- **설계의 품질 = 결과물의 품질**
