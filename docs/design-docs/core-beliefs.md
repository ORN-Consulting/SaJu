# core-beliefs.md — 프로젝트 근본 철학

> 이 프로젝트의 모든 설계 결정의 뿌리가 되는 핵심 원칙을 정의합니다.
> 에이전트가 판단이 모호한 상황에서 이 문서를 기준으로 결정합니다.

---

## 1. 규칙이 답이다

### 결정

사주 해석을 LLM(AI 생성)이 아닌 **전통 명리학 규칙을 코드로 직접 구현**한다.

### 근거

- **정확성**: 같은 사주를 넣으면 항상 같은 결과가 나와야 한다. LLM은 이를 보장하지 못한다.
- **투명성**: 왜 이런 결과가 나왔는지 로직으로 추적할 수 있어야 한다.
- **비용**: LLM API 호출 비용이 없으므로 무료 무제한 서비스가 가능하다.
- **신뢰**: "전통 명리학 기반"이라는 문구의 신뢰성을 실제 코드로 뒷받침한다.

### 적용 원칙

- 모든 사주 계산은 명리학 이론에 근거한 규칙으로 구현한다.
- "대략적인" 해석이나 확률적 판단은 하지 않는다.
- 규칙으로 표현할 수 없는 영역은 MVP에서 제외하고, 규칙화가 가능해지면 추가한다.

---

## 2. 단순함이 최선이다

### 결정

비회원 일회성 계산. DB 없음. 서버 사이드 로직 없음.

### 근거

- **진입 장벽 제거**: 회원가입은 사용자 이탈의 가장 큰 원인이다.
- **운영 비용 제로**: DB가 없으면 관리할 것이 없다. Vercel 무료 티어로 충분하다.
- **개인정보 부담 없음**: 생년월일시는 개인정보에 해당할 수 있다. 저장하지 않으면 문제가 없다.
- **개발 속도**: 인증, 세션, CRUD를 빼면 핵심 기능에만 집중할 수 있다.

### 적용 원칙

- 서버에 상태를 저장하지 않는다. 모든 계산은 클라이언트에서 완결된다.
- 새 기능을 추가할 때 DB가 필수인지 반드시 재검토한다. DB 없이 해결할 수 있으면 DB 없이 한다.
- 추후 회원 기능이 필요해지면, 기존 비회원 기능을 유지한 채 선택적으로 추가한다.

---

## 3. 접근성 > 정밀도 (MVP 단계)

### 결정

MVP에서는 사주 분석의 정밀도보다 **접근성과 이해 용이성**을 우선한다.

### 근거

- 명리학은 깊이가 무한하다. 완벽한 분석을 목표로 하면 영원히 출시할 수 없다.
- 대부분의 사용자는 명리학 전문가가 아니다. 쉽게 이해할 수 있는 결과가 더 가치 있다.
- 지장간 본기만, 억부법 기본 등의 단순화는 MVP에서 합리적이다.

### 적용 원칙

- 복잡한 이론(조후법, 통관법, 격국론 등)은 MVP 이후로 미룬다.
- 해석 텍스트는 전문 용어를 쓰되, 반드시 쉬운 설명을 병기한다.
- "정확하지 않을 수 있음"을 솔직하게 안내한다 (절기 경계, 3주 분석 등).

---

## 4. 무료는 진짜 무료다

### 결정

모든 기능을 무료로 제공한다. 횟수 제한, 유료 업그레이드, 광고 없음.

### 근거

- 사주 앱 시장에 유료 장벽이나 광고가 난무한다. 진짜 무료는 차별점이 된다.
- 서버 비용이 거의 없으므로 (클라이언트 계산 + Vercel 무료) 실현 가능하다.
- 수익 모델은 MVP 이후에 고민해도 늦지 않다.

### 적용 원칙

- "무료 분석 3회 후 결제" 같은 구조를 만들지 않는다.
- 광고 삽입을 위한 공간이나 로직을 미리 설계하지 않는다.
- 수익화가 필요해지면 부가 서비스(프리미엄 해석, 상담 연결 등)로 접근하되, 기본 분석은 무료를 유지한다.

---

## 5. 판단 기준 요약

에이전트가 설계 결정이 필요한 순간, 아래 우선순위로 판단합니다:

1. **사용자 편의** — 복잡하면 빼라
2. **규칙 기반 정확성** — 추측하지 말고, 규칙이 없으면 구현하지 마라
3. **운영 단순성** — 관리할 것을 늘리지 마라
4. **확장 가능성** — 지금 안 만들되, 나중에 붙일 수 있게 해라
