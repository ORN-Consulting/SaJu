# new-user-onboarding.md — 랜딩 / 첫 진입 화면 명세

> 사용자가 앱에 처음 진입했을 때 보는 화면의 기능 명세입니다.
> 단일 페이지(`app/page.tsx`)의 최상단 영역으로, 입력 폼(`BirthInputForm`) 위에 위치합니다.

---

## 1. 개요

| 항목 | 내용 |
|------|------|
| **위치** | 메인 페이지 최상단 |
| **역할** | 앱 소개 + 입력 폼으로 유도 |
| **분위기** | 깨끗하고 모던한 느낌 |
| **구성** | 헤더 → 히어로 섹션 → 특징 소개 3개 → 입력 폼(별도 명세) |

---

## 2. 페이지 전체 흐름

```
┌──────────────────────────────────────┐
│  [로고]                    [네비]     │  ← 헤더
├──────────────────────────────────────┤
│                                      │
│          히어로 섹션                   │  ← 헤드라인 + CTA
│                                      │
├──────────────────────────────────────┤
│                                      │
│       특징 소개 (3개 카드)             │
│                                      │
├──────────────────────────────────────┤
│                                      │
│       입력 폼 (birth-input.md)        │  ← CTA 클릭 시 여기로 스크롤
│                                      │
├──────────────────────────────────────┤
│                                      │
│       결과 영역 (saju-result.md)      │  ← 분석 후 표시
│                                      │
├──────────────────────────────────────┤
│           푸터                        │
└──────────────────────────────────────┘
```

## 3. 헤더

| 속성 | 내용 |
|------|------|
| **컴포넌트** | `Header.tsx` |
| **위치** | 페이지 최상단, 고정(sticky) 또는 일반 |
| **높이** | 56~64px |

### 3.1 헤더 구성

```
┌──────────────────────────────────────┐
│  [🔮 앱이름]              [소개] [문의]│
└──────────────────────────────────────┘
```

#### 왼쪽: 로고 + 앱 이름
- 텍스트 로고 (아이콘 + 앱 이름)
- 앱 이름은 미정 → 에이전트가 플레이스홀더로 구현 (예: "사주풀이")
- 클릭 시 페이지 최상단으로 스크롤

#### 오른쪽: 네비게이션
- MVP에서는 최소한의 링크만:
  - "소개" → 특징 소개 섹션으로 스크롤
  - "분석하기" → 입력 폼으로 스크롤
- 추후 확장 시: "사주란?", "오행 알아보기" 등 SEO 페이지 링크 추가

### 3.2 모바일 헤더

- 네비게이션 링크가 2개 이하이면 그대로 표시
- 3개 이상이 되면 햄버거 메뉴로 전환

## 4. 히어로 섹션

| 속성 | 내용 |
|------|------|
| **컴포넌트** | `HeroSection.tsx` |
| **높이** | 뷰포트 60~70% 정도 (전체 화면 X) |
| **배경** | 깔끔한 단색 또는 미묘한 그라데이션 |

### 4.1 구성 요소

```
┌──────────────────────────────────────┐
│                                      │
│                                      │
│     당신의 사주를 알아보세요             │  ← 헤드라인
│                                      │
│  전통 명리학에 기반한 사주팔자 분석,      │  ← 서브텍스트
│  지금 바로 무료로 확인하세요.            │
│                                      │
│        [ 사주 분석 시작하기 ]           │  ← CTA 버튼
│                                      │
│              ↓                       │  ← 스크롤 유도 (선택)
│                                      │
└──────────────────────────────────────┘
```

#### 헤드라인

| 속성 | 내용 |
|------|------|
| **텍스트** | "당신의 사주를 알아보세요" (예시, 에이전트 조정 가능) |
| **스타일** | 큰 볼드 텍스트 (text-4xl ~ text-5xl) |
| **색상** | 다크 텍스트 (모던한 느낌 유지) |

#### 서브텍스트

| 속성 | 내용 |
|------|------|
| **텍스트** | "전통 명리학에 기반한 사주팔자 분석, 지금 바로 무료로 확인하세요." |
| **스타일** | 중간 크기 (text-lg ~ text-xl), 회색 계열 |
| **역할** | 앱의 핵심 가치를 한 문장으로 전달 |

#### CTA 버튼

| 속성 | 내용 |
|------|------|
| **텍스트** | "사주 분석 시작하기" |
| **스타일** | 프라이머리 색상, 충분한 크기 (px-8 py-3 이상) |
| **동작** | 클릭 → 입력 폼 섹션으로 smooth scroll |
| **호버** | 살짝 밝아지거나 그림자 추가 |

#### 스크롤 유도 (선택)

- CTA 아래에 작은 화살표(↓) 또는 "아래로 스크롤" 힌트
- 미세한 바운스 애니메이션
- 사용자가 스크롤하면 자연스럽게 사라짐
- 필수는 아님 — 에이전트 판단에 따라 포함 여부 결정

## 5. 특징 소개 (3개 카드)

| 속성 | 내용 |
|------|------|
| **컴포넌트** | `FeatureCards.tsx` |
| **레이아웃** | 3열 카드 (데스크톱), 1열 세로 배치 (모바일) |
| **위치** | 히어로 섹션 아래, 입력 폼 위 |

### 5.1 카드 내용

#### 카드 1: 전통 명리학 기반

| 속성 | 내용 |
|------|------|
| **아이콘** | 📚 또는 적절한 아이콘 |
| **제목** | "전통 명리학 기반 분석" |
| **설명** | "수천 년간 이어져 온 명리학 이론을 규칙으로 구현했어요. AI 추측이 아닌 전통 이론에 따른 정확한 분석을 제공해요." |

#### 카드 2: 가입 없이 바로

| 속성 | 내용 |
|------|------|
| **아이콘** | ⚡ 또는 적절한 아이콘 |
| **제목** | "가입 없이 바로 분석" |
| **설명** | "회원가입이나 로그인 없이 생년월일시만 입력하면 바로 결과를 확인할 수 있어요." |

#### 카드 3: 무료 무제한

| 속성 | 내용 |
|------|------|
| **아이콘** | 🎁 또는 적절한 아이콘 |
| **제목** | "완전 무료, 무제한 이용" |
| **설명** | "횟수 제한이나 유료 기능 없이 모든 분석을 무료로 이용할 수 있어요." |

### 5.2 카드 스타일

- 깔끔한 카드 형태 (테두리 또는 살짝 올라온 그림자)
- 아이콘은 상단 중앙 또는 좌측
- 제목: 굵은 텍스트 (text-lg, bold)
- 설명: 회색 계열 본문 (text-sm ~ text-base)
- 호버 시 살짝 올라오는 효과 (선택)

### 5.3 텍스트 가이드라인

- 제목: 격식체 (명사형 종결)
- 설명: 친근체 (~이에요, ~해요)
- 위 텍스트는 예시이며, 에이전트가 톤에 맞게 조정 가능

## 7. 애니메이션

### 7.1 첫 로딩 시

| 요소 | 애니메이션 | 딜레이 |
|------|----------|--------|
| 헤더 | 위에서 슬라이드 다운 | 0ms |
| 헤드라인 | 페이드인 | 200ms |
| 서브텍스트 | 페이드인 | 400ms |
| CTA 버튼 | 페이드인 + 살짝 스케일업 | 600ms |
| 특징 카드 3개 | 아래에서 페이드인 (순차) | 800ms, 900ms, 1000ms |

### 7.2 스크롤 기반 (선택)

- 특징 카드가 뷰포트에 진입할 때 페이드인 (Intersection Observer)
- 과도한 애니메이션은 지양 — 모던하고 깔끔한 느낌 유지
- `prefers-reduced-motion` 미디어 쿼리 대응

- 전체 콘텐츠 중앙 정렬
- 최대 너비 제한 (1200px)
- 히어로: 넉넉한 상하 패딩
- 특징 카드: 3열, 균등 분배

## 9. 접근성

- 헤드라인은 `<h1>` 태그 사용 (페이지당 1개)
- 특징 카드 제목은 `<h2>` 또는 `<h3>`
- CTA 버튼에 명확한 `aria-label` (예: "사주 분석 입력 폼으로 이동")
- 네비게이션 링크에 `aria-current` 속성
- 아이콘은 장식적 요소로 `aria-hidden="true"` 처리
- 스크롤 유도 화살표는 스크린 리더에서 숨김

MVP에서는 메인 페이지 1개이므로, 이 페이지의 메타데이터를 잘 설정합니다.

```typescript
    type: 'website',
  },
};
```

## 11. 브랜딩 참고사항

앱 이름은 미정입니다. 에이전트가 개발 시 플레이스홀더를 사용하되, 쉽게 교체할 수 있도록 **상수로 관리**합니다:

```typescript
// lib/constants.ts 또는 lib/brand.ts
export const BRAND = {
  name: '사주풀이',        // 플레이스홀더 — 추후 변경
  tagline: '당신의 사주를 알아보세요',
  emoji: '🔮',            // 로고 대용 이모지
};
```

- 앱 이름이 등장하는 모든 곳에서 이 상수를 참조
- 이름 확정 시 한 곳만 수정하면 전체 반영
