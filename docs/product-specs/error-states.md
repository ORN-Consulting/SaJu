# error-states.md — 에러 상황별 처리 명세

> 사주 분석 앱에서 발생할 수 있는 에러 상황과 사용자 안내 방법을 정의합니다.
> 모든 에러 메시지는 친근체로 작성하며, 에러 원인 + 해결 방법을 함께 안내합니다.

---

## 1. 에러 표시 원칙

| 항목 | 내용 |
|------|------|
| **표시 방식** | 입력 필드 아래에 인라인 메시지 |
| **안내 수준** | 에러 원인 + 해결 방법 |
| **문체** | 친근체 (~이에요, ~해요, ~해 주세요) |
| **스타일** | 빨간 계열 텍스트 + 아이콘(⚠️ 또는 ❗) |
| **접근성** | `aria-live="polite"` + `role="alert"` |

### 인라인 에러 메시지 형태

```
┌──────────────────────┐
│ 월 선택           ▼  │
└──────────────────────┘
⚠️ 해당 음력 날짜는 존재하지 않아요.    ← 에러 원인
   양력으로 입력하거나 날짜를 확인해 주세요.  ← 해결 방법
```

---

## 2. 입력 단계 에러

### 2.1 필수 항목 미입력

분석 버튼 클릭 시 필수 항목이 비어있을 때 발생합니다.

| 미입력 항목 | 에러 메시지 |
|------------|-----------|
| 연도 | "연도를 선택해 주세요." |
| 월 | "월을 선택해 주세요." |
| 일 | "일을 선택해 주세요." |
| 생시 | "시간을 선택해 주세요. 모르시면 '모름'을 선택할 수 있어요." |
| 성별 | "성별을 선택해 주세요." |

**동작:**
- 미입력 필드에 빨간 테두리 + 인라인 에러 메시지 표시
- 첫 번째 미입력 필드로 포커스 이동
- 분석 버튼은 비활성 상태 유지 (필수 항목 모두 입력 전까지)

### 2.2 날짜 유효성 에러

#### 미래 날짜

| 항목 | 내용 |
|------|------|
| **조건** | 입력된 날짜가 오늘 이후 |
| **메시지** | "미래 날짜는 분석할 수 없어요. 생년월일을 다시 확인해 주세요." |
| **검증 시점** | 분석 버튼 클릭 시 |

#### 존재하지 않는 양력 날짜

| 항목 | 내용 |
|------|------|
| **조건** | 2월 30일, 4월 31일 등 |
| **메시지** | "존재하지 않는 날짜예요. 일(日)을 다시 확인해 주세요." |
| **검증 시점** | 일(日) 선택 즉시 |
| **예방** | 연/월 선택에 따라 일(日) 드롭다운 범위를 동적 조정 |

#### 존재하지 않는 음력 날짜

| 항목 | 내용 |
|------|------|
| **조건** | 해당 음력 연/월에 존재하지 않는 날짜 (예: 작은달에 30일) |
| **메시지** | "해당 음력 날짜는 존재하지 않아요. 날짜를 확인하거나 양력으로 입력해 주세요." |
| **검증 시점** | 일(日) 선택 즉시 (음력 모드일 때) |

#### 존재하지 않는 윤달

| 항목 | 내용 |
|------|------|
| **조건** | 해당 연/월에 윤달이 없는데 윤달 체크 |
| **메시지** | "[YYYY]년 [MM]월에는 윤달이 없어요. 윤달 체크를 해제하거나 날짜를 확인해 주세요." |
| **검증 시점** | 윤달 체크 즉시 |

### 2.3 지원 범위 초과

| 항목 | 내용 |
|------|------|
| **조건** | 1920년 이전 또는 라이브러리 지원 범위 밖 |
| **메시지** | "지원 가능한 범위는 1920년부터예요. 연도를 다시 확인해 주세요." |
| **검증 시점** | 연도 선택 즉시 |
| **예방** | 드롭다운에 1920년 이전 연도를 포함하지 않음 |

## 3. 계산 단계 에러

클라이언트 사이드 계산 중 예외가 발생하는 경우입니다. 드물지만 대비가 필요합니다.

### 3.1 만세력 계산 실패

| 항목 | 내용 |
|------|------|
| **조건** | `lunar-javascript` 라이브러리 내부 에러 |
| **메시지** | "사주 계산 중 문제가 발생했어요. 날짜를 다시 확인하거나, 잠시 후 다시 시도해 주세요." |
| **표시 위치** | 결과 영역 위치에 에러 카드 표시 |
| **추가 동작** | `console.error`로 상세 에러 기록 (개발 환경) |

### 3.2 절기 경계 계산 오류

| 항목 | 내용 |
|------|------|
| **조건** | 절기 경계 시간대에서 월주 판정이 모호한 경우 |
| **메시지** | "절기 경계에 해당하는 날짜예요. 결과가 정확하지 않을 수 있어요." |
| **표시 위치** | 결과 상단에 안내 배너 (에러가 아닌 주의 안내) |
| **스타일** | 노란 계열 배경 (warning, 에러가 아님) |

---

## 4. 해석 단계 에러

### 4.1 해석 텍스트 매핑 누락

| 항목 | 내용 |
|------|------|
| **조건** | 특정 십성/용신 조합에 대한 해석 텍스트가 JSON에 없는 경우 |
| **처리** | fallback 텍스트 표시 |
| **fallback** | "[십성명]에 해당해요. 자세한 해석은 준비 중이에요." |
| **표시** | 정상 결과와 동일한 위치에 fallback 텍스트 표시 |

> 사용자에게 에러로 보이지 않도록 자연스럽게 처리.
> 빈 공간이나 undefined가 노출되어서는 안 됨.

## 5. 에러 카드 (계산 실패 시)

정상적인 결과 대신 표시되는 에러 전용 화면입니다.

```
┌──────────────────────────────────────┐
│                                      │
│              ⚠️                      │
│                                      │
│    사주 계산 중 문제가 발생했어요.       │
│                                      │
│    날짜를 다시 확인하거나,              │
│    잠시 후 다시 시도해 주세요.          │
│                                      │
│       [ 다시 분석하기 ]               │
│                                      │
└──────────────────────────────────────┘
```

| 속성 | 내용 |
|------|------|
| **컴포넌트** | `ErrorCard.tsx` |
| **위치** | 결과 영역 위치 (정상 결과 대신) |
| **스타일** | 중앙 정렬, 회색 배경, 부드러운 테두리 |
| **버튼** | "다시 분석하기" → 입력 폼으로 스크롤 |

## 6. 주의 배너 (경고, 에러 아님)

에러는 아니지만 사용자에게 알려줘야 할 상황입니다.

### 6.1 절기 경계 안내

```
┌──────────────────────────────────────┐
│ ℹ️ 절기 경계에 해당하는 날짜예요.       │
│    결과가 정확하지 않을 수 있어요.       │
└──────────────────────────────────────┘
```

### 6.2 3주 분석 안내 (생시 모름)

```
┌──────────────────────────────────────┐
│ ℹ️ 태어난 시간을 모르면 시주(時柱) 없이  │
│    년·월·일 기준으로 분석해요.           │
└──────────────────────────────────────┘
```

| 속성 | 내용 |
|------|------|
| **스타일** | 파란 또는 노란 계열 배경 (info/warning) |
| **위치** | 결과 영역 상단 (결과 콘텐츠 위) |
| **닫기** | 닫기(X) 버튼 선택 제공 |

## 8. 에러 처리 구현 가이드

### 8.1 입력 검증 패턴

```typescript
// 각 입력 필드에 에러 상태를 관리
interface FieldError {
  code: string;     // E001, E002, ...
  message: string;  // 사용자에게 보여줄 메시지
  solution: string; // 해결 방법 안내 (없으면 빈 문자열)
}

// 에러가 있으면 필드 아래에 인라인 표시
// 에러가 없으면 아무것도 표시하지 않음
```

### 8.2 계산 에러 패턴

```typescript
// lib/saju/index.ts 의 계산 함수는 try-catch로 감싸기
try {
  const result = calculateSaju(input);
  return { success: true, data: result };
} catch (error) {
  console.error('[E011] 사주 계산 실패:', error);
  return { success: false, error: 'E011' };
}
```

### 8.3 에러 메시지 상수 관리

```typescript
// lib/errors.ts
export const ERROR_MESSAGES: Record<string, { message: string; solution: string }> = {
  E001: { message: '연도를 선택해 주세요.', solution: '' },
  E006: { message: '미래 날짜는 분석할 수 없어요.', solution: '생년월일을 다시 확인해 주세요.' },
  // ...
};
```

> 에러 메시지를 코드에 하드코딩하지 않고, 상수 파일에서 관리합니다.
