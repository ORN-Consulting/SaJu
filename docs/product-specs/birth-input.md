# birth-input.md — 생년월일시 입력 화면 명세

> 사용자가 사주 분석을 위해 생년월일시를 입력하는 화면의 기능 명세입니다.
> 이 화면은 앱의 메인 페이지(`app/page.tsx`)에 포함되며, 결과 화면과 같은 페이지에 존재합니다.

---

## 1. 개요

| 항목 | 내용 |
|------|------|
| **위치** | 메인 페이지 상단 |
| **컴포넌트** | `BirthInputForm.tsx` |
| **역할** | 생년월일시 + 성별 입력 → 사주 분석 트리거 |
| **결과 전환** | 분석 버튼 클릭 → 스크롤 애니메이션으로 결과 영역 이동 |

---

## 2. 입력 항목

### 2.1 역법 선택 (양력/음력)

| 속성 | 내용 |
|------|------|
| **UI** | 토글 버튼 또는 세그먼트 컨트롤 (양력 / 음력) |
| **기본값** | 양력 |
| **동작** | 음력 선택 시 윤달 체크박스 표시 |

**음력 선택 시 추가 UI:**
- 윤달 여부 체크박스: "윤달" (기본: 체크 해제)
- 안내 텍스트: "음력 날짜를 모르시면 양력으로 입력해 주세요"

### 2.2 날짜 입력 (셀렉트박스)

세 개의 드롭다운으로 구성합니다:

#### 연도 (年)
| 속성 | 내용 |
|------|------|
| **UI** | 드롭다운 셀렉트박스 |
| **범위** | 1920년 ~ 현재 연도 |
| **기본값** | 1990년 (또는 빈 상태 "연도 선택") |
| **정렬** | 최신 연도부터 내림차순 |

#### 월 (月)
| 속성 | 내용 |
|------|------|
| **UI** | 드롭다운 셀렉트박스 |
| **범위** | 1월 ~ 12월 |
| **기본값** | 빈 상태 ("월 선택") |

#### 일 (日)
| 속성 | 내용 |
|------|------|
| **UI** | 드롭다운 셀렉트박스 |
| **범위** | 연/월에 따라 동적 변경 |
| **기본값** | 빈 상태 ("일 선택") |

**일(日) 범위 규칙:**
- 양력: 해당 연/월의 실제 일수 (윤년 2월 = 29일)
- 음력: 해당 연/월의 음력 일수 (큰달 30일, 작은달 29일)
- 연/월 미선택 시: 1~31일 표시

### 2.3 생시 입력

기본은 **시진(時辰) 선택**, 토글로 **정확한 시간 입력** 전환 가능합니다.

#### 모드 A: 시진 선택 (기본)

| 속성 | 내용 |
|------|------|
| **UI** | 드롭다운 셀렉트박스 |
| **기본값** | 빈 상태 ("시간 선택") |

**시진 목록:**

| 시진 | 시간 범위 | 표시 텍스트 |
|------|----------|------------|
| 자시(子時) | 23:00 ~ 01:00 | 자시 (23:00~01:00) |
| 축시(丑時) | 01:00 ~ 03:00 | 축시 (01:00~03:00) |
| 인시(寅時) | 03:00 ~ 05:00 | 인시 (03:00~05:00) |
| 묘시(卯時) | 05:00 ~ 07:00 | 묘시 (05:00~07:00) |
| 진시(辰時) | 07:00 ~ 09:00 | 진시 (07:00~09:00) |
| 사시(巳時) | 09:00 ~ 11:00 | 사시 (09:00~11:00) |
| 오시(午時) | 11:00 ~ 13:00 | 오시 (11:00~13:00) |
| 미시(未時) | 13:00 ~ 15:00 | 미시 (13:00~15:00) |
| 신시(申時) | 15:00 ~ 17:00 | 신시 (15:00~17:00) |
| 유시(酉時) | 17:00 ~ 19:00 | 유시 (17:00~19:00) |
| 술시(戌時) | 19:00 ~ 21:00 | 술시 (19:00~21:00) |
| 해시(亥時) | 21:00 ~ 23:00 | 해시 (21:00~23:00) |
| **모름** | — | 모름 (시주 제외 분석) |

#### 모드 B: 정확한 시간 입력

| 속성 | 내용 |
|------|------|
| **전환** | "정확한 시간을 알고 있어요" 텍스트 링크 클릭 시 전환 |
| **UI** | 시(0~23) 드롭다운 + 분(00, 10, 20, 30, 40, 50) 드롭다운 |
| **복귀** | "시진으로 선택하기" 링크로 모드 A 복귀 |

**정확한 시간 → 시진 자동 매핑:**
- 입력된 시:분을 기반으로 내부적으로 시진을 결정
- 야자시 규칙 적용: 23:00~23:59는 당일 자시

#### "모름" 선택 시 동작

- 시진 셀렉트에서 "모름" 선택 가능
- 선택 시 안내 메시지 표시: "태어난 시간을 모르면 년·월·일 기준으로 분석해요. 시주(時柱)는 결과에서 제외돼요."
- 분석 시 3주(년주·월주·일주)만 산출, 시주 관련 결과는 표시하지 않음

### 2.4 성별 입력

| 속성 | 내용 |
|------|------|
| **UI** | 세그먼트 컨트롤 또는 라디오 버튼 (남성 / 여성) |
| **기본값** | 미선택 |
| **필수 여부** | 필수 |
| **용도** | 현재 MVP에서는 표시용, 향후 대운 순행/역행 결정에 사용 |

## 3. 분석 버튼

| 속성 | 내용 |
|------|------|
| **텍스트** | "사주 분석하기" |
| **위치** | 입력 폼 하단 |
| **상태** | 필수 항목 미입력 시 비활성화 (disabled) |
| **동작** | 클릭 → 사주 계산 → 스크롤 애니메이션으로 결과 영역 이동 |

### 3.1 필수 입력 조건

분석 버튼이 활성화되려면 아래 조건을 모두 충족해야 합니다:

- [x] 연도 선택 완료
- [x] 월 선택 완료
- [x] 일 선택 완료
- [x] 생시 선택 완료 (시진 선택 또는 정확한 시간 입력 또는 "모름")
- [x] 성별 선택 완료

### 3.2 분석 실행 흐름

```
버튼 클릭
  → 입력값 유효성 최종 검증
  → 사주 계산 실행 (lib/saju/index.ts)
  → 결과 상태 업데이트 (React state)
  → 스크롤 애니메이션으로 결과 영역 이동 (smooth scroll)
  → 결과 영역 표시
```

---

## 4. 유효성 검사

### 4.1 실시간 검증 (입력 즉시)

| 검증 항목 | 규칙 | 에러 메시지 |
|----------|------|-----------|
| 연도 범위 | 1920~현재 연도 | "지원 범위는 1920년부터입니다" |
| 음력 날짜 유효성 | 해당 음력 연/월에 존재하는 날짜인지 | "해당 음력 날짜는 존재하지 않아요" |
| 윤달 유효성 | 해당 연/월에 윤달이 있는지 | "해당 연도/월에는 윤달이 없어요" |

### 4.2 제출 시 검증 (버튼 클릭 시)

| 검증 항목 | 규칙 | 에러 메시지 |
|----------|------|-----------|
| 미래 날짜 | 오늘 이후 날짜 차단 | "미래 날짜는 입력할 수 없어요" |
| 필수 항목 누락 | 모든 필수 항목 입력 확인 | 해당 필드 하이라이트 |

---

## 5. 결과 전환 인터랙션

### 5.1 스크롤 애니메이션

```
분석 버튼 클릭
  → 결과 영역이 뷰포트에 보이도록 smooth scroll
  → 스크롤 완료 후 결과 영역에 fade-in 애니메이션
  → 소요 시간: 약 500~800ms
```

### 5.2 재분석

- 결과가 표시된 상태에서 입력값을 변경하면 결과 영역이 사라지거나 흐려짐
- 다시 "사주 분석하기" 버튼을 눌러야 새 결과 표시
- 또는 결과 영역 상단에 "다시 분석하기" 버튼 제공

## 6. 반응형 레이아웃

### 모바일 (< 640px)
- 모든 입력 필드 세로 배치 (1열)
- 셀렉트박스는 전체 너비
- 시진 목록은 시간 범위 텍스트 생략 가능: "자시" (23~01)

### 태블릿 (640px ~ 1024px)
- 연/월/일을 한 줄에 배치 (3열)
- 생시, 성별은 아래 줄

### 데스크톱 (> 1024px)
- 입력 폼을 카드 형태로 중앙 정렬
- 최대 너비 제한 (640px 정도)

---

## 7. 접근성

- 모든 셀렉트박스에 `label` 연결
- 키보드 탐색 가능 (Tab으로 이동, Enter로 선택)
- 에러 메시지는 `aria-live="polite"`로 스크린 리더 전달
- 색상만으로 정보를 전달하지 않음 (에러 상태는 텍스트 + 아이콘)
- 버튼 비활성화 시 이유를 툴팁 또는 안내 텍스트로 제공
